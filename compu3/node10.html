Uso avanzado

Si el usuario desea manipular algunos items aparte de ``Control.cpp", podría ser necesario tener que ejecutar qmake para generar un nuevo Makefile (ver apartado 2.3), si ésto se va a dar más de una vez sería conveniente hacer un enlace en usr/bin a qmake, para no tener que hacer el export cada vez que desee emplear qmake:

$ sudo ln -s RUTAQMAKE/qmake /usr/bin/qmake

Donde en el caso de instalación por defecto de QT:

$ sudo ln -s $HOME/Qt5.2.1/5.2.1/gcc/bin/qmake /usr/bin/qmake
Para la modificación del código de control de usuario ver sección 7.
